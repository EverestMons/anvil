Full text enables search without file I/O but increases DB size. (2) Should `chunk_fingerprints` be a separate table or columns on `code_chunks`? Separate table follows study's pattern and keeps the core table lean. (3) Index strategy — which columns need indexes beyond PKs and FKs? At minimum: `code_chunks(project_id, file_path)`, `code_chunks(chunk_type)`, `chunk_symbol_bindings(symbol_name)`, `chunk_dependencies(source_chunk_id)`, `chunk_dependencies(target_chunk_id)`, `git_changes(project_id, file_path)`, `health_scores(chunk_id, cycle_id)`. **Include a "How to verify this was implemented correctly" section** with PRAGMA queries the QA agent should run. **Deposit:** `anvil/knowledge/architecture/schema-blueprint-2026-03-29.md` using `with open("/Users/marklehn/Desktop/GitHub/anvil/knowledge/architecture/schema-blueprint-2026-03-29.md", "w") as f: f.write(content)`.
