## Flag: Mission Context Injection — "Given that" Not Present

**Root cause:** `_extract_project_mission()` in `src/lab.py` looks for headings `## Mission`, `## Overview`, or `## Purpose`. invoice-pulse's `PROJECT_BRIEF.md` uses `## What This Project Is` — none of the expected headings match. The function returns an empty string, causing all 20 `what_needs_discovering` fields to fall back to the non-mission template:

> "Does '{name}' perform logic critical to the project's core goals (as described in PROJECT_BRIEF)?"

This means QA's `grep "Given that"` check (Step 2) will find zero matches. The findings still reference PROJECT_BRIEF conceptually, but the mission sentence is not injected.

**Impact:** QA Step 2 deliverable (3) will fail: `grep for 'Given that' — assert at least one match`.

**Action required from CEO:** Confirm whether (a) the heading pattern in `_extract_project_mission` should be expanded to include `## What This Project Is`, or (b) the QA grep check should be updated to match the actual fallback text.
